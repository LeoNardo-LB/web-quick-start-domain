# INFRASTRUCTURE LAYER

Technical implementations - repository adapters, external service integrations, cross-cutting concerns.

## STRUCTURE
```
infrastructure/
├── bizshared/       # Shared infrastructure
│   ├── dal/         # MyBatis-Flex generated code
│   │   └── generated/ # Mapper interfaces, entity DOs
│   ├── event/       # Event publishing/consuming
│   ├── client/      # Client cache/notification implementations
│   └── util/        # Utilities (SpringContext, Kryo)
├── common/          # Reusable technical services
│   ├── cache/       # CacheClient implementations (Caffeine/Redis)
│   ├── file/        # OSS service (Local/RustFS)
│   ├── log/         # Log persistence
│   ├── notification/ # SMS/Email clients (Aliyun)
│   └── search/      # Elasticsearch service
└── example/         # Example implementations
    └── persistence/ # OrderAggrRepositoryImpl
```

## WHERE TO LOOK
| Task | Location | Notes |
|------|----------|-------|
| Repository Impls | infrastructure/**/persistence/*RepositoryImpl.java | MyBatis-Flex + converters |
| Mappers | infrastructure/**/dal/generated/mapper/ | Generated by MyBatis-Flex |
| Event Publisher | infrastructure/bizshared/event/ | SpringEventPublisherImpl, KafkaEventPublisherImpl |
| Cache Services | infrastructure/common/cache/ | CaffeineCacheClientImpl, RedisCacheClientImpl |
| OSS Service | infrastructure/common/file/ | LocalOssServiceImpl, RustFsOssServiceImpl |

## CONVENTIONS
- Implements interfaces defined in domain/
- Auto-detection: `@ConditionalOnBean` for Redis/Kafka/RustFS
- MapStruct converters: `*BusinessConverter` for DO ↔ Domain mapping
- MyBatis-Flex generates DO classes in `dal/generated/entity/`
- Kryo for serialization (domain event persistence)

## ANTI-PATTERNS
- ❌ Business logic in infrastructure (delegate to domain)
- ❌ Direct access to domain models from adapters (use app layer)
- ❌ Configuration classes (must be in start/)
- ❌ Repository interfaces (define in domain/)
