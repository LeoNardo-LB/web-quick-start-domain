# AI工作记录使用指南

> 本目录存储AI辅助开发的完整活动记录
> 最后更新：2026-01-15

---

## 📝 目录说明

本目录按DDD分包组织，每个模块一个独立的日志文件：

```
dev_log/
├── README.md                # 本使用指南
├── orderdev_log.md         # 订单模块开发日志
├── searchdev_log.md        # 搜索模块开发日志
├── paymentdev_log.md       # 支付模块开发日志
└── [module]dev_log.md      # 其他模块日志
```

---

## 🎯 记录内容

每条记录包含：

1. **🎯 需求描述**：用户的原始需求
2. **📋 实现计划**：初步计划和演进过程
3. **🔧 实现细节**：新增/修改的文件清单、关键决策
4. **🔄 方案演进**：用户反馈导致的调整
5. **✅ 验证结果**：测试通过率、覆盖率等量化指标
6. **📦 交付物**：代码文件、测试文件统计
7. **🤔 复盘总结**：亮点、待改进、经验沉淀

---

## 📅 时间规则

**关键规则**：

- ⏰ **按天划分**：以每天凌晨4点为分界点
- ✅ **当天可编辑**：同一天内可追加、修改当天内容
- ❌ **历史不可改**：历史日期内容只读，不可修改
- 📌 **自动追加**：AI自动生成并追加到对应模块日志文件

**示例**：

```markdown
## 2026-01-15

### 📌 [需求1] 订单快照功能
**时间**: 14:30 - 16:00

[当天可编辑的内容...]

---

## 2026-01-14

### 📌 [需求1] 库存锁定功能
**时间**: 09:00 - 12:00

> ⚠️ **历史锁定**：此日期的内容已锁定，不可修改
> 如需补充，请在当天新增"补充记录"章节

[历史内容...]
```

## 🔍 检索技巧

### 按标签查找

```bash
# 查找所有新功能记录
grep "#新功能" *.md

# 查找订单模块的优化记录
grep "#优化" orderdev_log.md
```

### 按日期查找

```bash
# 查找特定日期的记录
grep "## 2026-01-15" *.md
```

### 按文件查找

```bash
# 查找涉及某文件的所有记录
grep "OrderSnapshot.java" *.md
```

### 按模块查找

```bash
# 查看订单模块的所有记录
cat orderdev_log.md

# 查看最近7天的记录
head -100 orderdev_log.md
```

## 📋 记录模板

AI在生成记录时，应遵循以下模板：

```markdown
## YYYY-MM-DD

### 📌 [需求N] {需求标题}

**时间**: HH:MM - HH:MM
**标签**: `#类型` `#关键词1` `#关键词2`

#### 🎯 需求描述
[保留用户的原始需求描述]

#### 📋 实现计划
- [x] 计划项1（已完成）
- [x] 计划项2（已完成）
- [ ] 计划项3（未完成）

#### 🔧 实现细节

**新增文件** (N个):
- path/to/File1.java - 功能说明
- path/to/File2.java - 功能说明

**修改文件** (N个):
- path/to/File3.java - 修改说明

**关键设计决策**:
- ✨ 决策点1：原因说明
- ✨ 决策点2：原因说明

#### 🔄 方案演进

**时间** - 用户反馈/问题发现
- 调整：具体改动内容

#### ✅ 验证结果
- ✅ 单元测试通过率：100% (N/N)
- ✅ 启动测试通过
- ✅ 测试覆盖率：XX%

#### 📦 交付物
- 代码文件：N个（N新增 + N修改）
- 测试文件：N个
- 测试用例：N个

#### 🤔 复盘总结

**亮点**:
- 做得好的地方

**待改进**:
- 需要改进的地方

**经验沉淀**:
> 可复用于未来需求的宝贵经验

---
```

## 💡 使用建议

### 对于开发者

1. **定期回顾**：每周查看自己的开发日志，总结经验
2. **搜索复用**：遇到类似需求时，先搜索历史记录
3. **知识沉淀**：将复盘总结中的经验整理成文档

### 对于AI

1. **完成需求后**：主动询问是否需要记录
2. **生成记录时**：遵循模板，确保完整性
3. **历史参考**：开发新功能前，先搜索相关模块的历史记录

### 对于团队

1. **交接参考**：新成员可查看历史记录了解项目演进
2. **决策追溯**：理解为什么做某个技术决策
3. **最佳实践**：从复盘总结中提取团队最佳实践
