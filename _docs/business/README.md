### 技术选型示例

#### 技术方案对比示例

**数据库选型**：

| 方案 | 优点 | 缺点 | 适用场景 | 成本 |
|------|------|------|---------|------|
| MySQL | 成熟稳定、开源 | 海量数据性能一般 | 中小规模应用 | 免费 |
| PostgreSQL | 强大的查询能力、支持复杂查询 | 运维相对复杂 | 复杂查询、数据分析 | 免费 |
| MongoDB | 灵活的数据模型、高扩展性 | 弱事务支持 | 非结构化数据、快速迭代 | 免费 |

**选型结论**: 选择 MySQL
- 理由：项目为传统电商系统，关系型数据模型更合适
- 数据量预估：中等规模（< 1000万条）

**缓存选型**：

| 方案 | 优点 | 缺点 | 适用场景 | 成本 |
|------|------|------|---------|------|
| Redis | 高性能、丰富的数据结构 | 内存限制、持久化需额外处理 | 热点数据缓存 | 免费 |
| Memcached | 简单高效、分布式 | 数据结构单一 | 简单KV缓存 | 免费 |

**选型结论**: 选择 Redis
- 理由：需要支持多种数据结构（String、Hash、List）

**消息队列选型**：

| 方案 | 优点 | 缺点 | 适用场景 | 成本 |
|------|------|------|---------|------|
| Kafka | 高吞吐、持久化、分布式 | 运维复杂、学习曲线陡 | 大规模消息流 | 免费 |
| RabbitMQ | 灵活的路由、易部署 | 吞吐量相对较低 | 中小规模消息队列 | 免费 |

**选型结论**: 选择 Kafka
- 理由：项目未来可能需要大数据流处理

**前端框架选型**：

| 方案 | 优点 | 缺点 | 适用场景 | 成本 |
|------|------|------|---------|------|
| React | 生态成熟、组件丰富 | 包体积较大 | 复杂单页应用 | 免费 |
| Vue | 轻量级、易上手 | 生态相对较小 | 快速迭代项目 | 免费 |

**选型结论**: 选择 Vue
- 理由：团队熟悉度、学习成本低

### 数据设计示例

#### 表结构SQL示例

**用户表（user）**：

```sql
CREATE TABLE `user` (
  `id` BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '用户ID',
  `username` VARCHAR(50) NOT NULL UNIQUE COMMENT '用户名',
  `password` VARCHAR(255) NOT NULL COMMENT '密码（加密）',
  `email` VARCHAR(100) NOT NULL UNIQUE COMMENT '邮箱',
  `phone` VARCHAR(20) COMMENT '手机号',
  `status` TINYINT NOT NULL DEFAULT 1 COMMENT '状态：1-正常，2-冻结',
  `create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `update_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  INDEX `idx_username` (`username`),
  INDEX `idx_email` (`email`),
  INDEX `idx_phone` (`phone`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';
```

**商品表（product）**：

```sql
CREATE TABLE `product` (
  `id` BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '商品ID',
  `name` VARCHAR(200) NOT NULL COMMENT '商品名称',
  `category_id` BIGINT NOT NULL COMMENT '分类ID',
  `price` DECIMAL(10,2) NOT NULL COMMENT '价格',
  `stock` INT NOT NULL DEFAULT 0 COMMENT '库存',
  `status` TINYINT NOT NULL DEFAULT 1 COMMENT '状态：1-上架，2-下架',
  `create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `update_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  INDEX `idx_category` (`category_id`),
  INDEX `idx_status` (`status`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='商品表';
```

**订单表（order）**：

```sql
CREATE TABLE `order` (
  `id` BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '订单ID',
  `order_no` VARCHAR(50) NOT NULL UNIQUE COMMENT '订单号',
  `user_id` BIGINT NOT NULL COMMENT '用户ID',
  `total_amount` DECIMAL(10,2) NOT NULL COMMENT '订单总额',
  `status` TINYINT NOT NULL DEFAULT 1 COMMENT '状态：1-待支付，2-已支付，3-已取消',
  `create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `update_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  INDEX `idx_user_id` (`user_id`),
  INDEX `idx_order_no` (`order_no`),
  INDEX `idx_status` (`status`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='订单表';
```

    "pageSize": 20
  }

#### 核心流程示例

**用户注册流程**：
```
1. 用户填写注册信息
   ↓
2. 前端校验（邮箱格式、密码强度）
   ↓
3. 调用注册API
   ↓
4. 后端验证（用户名唯一、邮箱唯一）
   ↓
5. 生成Token并返回
   ↓
6. 前端存储Token，跳转到首页
```

**关键点**：
- 密码使用 BCrypt 加密
- Token 使用 JWT 签名，有效期2小时
- 异步发送注册邮件（使用RabbitMQ）

**商品浏览流程**：
```
1. 用户浏览商品列表
   ↓
2. 首次访问：从数据库查询
   ↓
3. 再次访问：从Redis缓存读取
   ↓
4. 缓存未命中：查询数据库并写入Redis
   ↓
5. 返回商品列表（分页）
```

**关键点**：
- Redis 缓存策略：Key 为 `product:list:{page}:{pageSize}`
- 缓存过期时间：5分钟
- 热点商品预加载

**订单创建流程**：
```
1. 用户选择商品，点击购买
   ↓
2. 前端校验（库存、用户状态）
   ↓
3. 调用创建订单API
   ↓
4. 后端事务开始
   ↓
5. 扣减库存（乐观锁）
   ↓
6. 创建订单记录
   ↓
7. 事务提交
   ↓
8. 发送订单创建消息到Kafka
   ↓
9. 返回订单ID
```

**关键点**：
- 使用数据库乐观锁防止超卖
- Kafka 异步处理后续流程（发货、通知）
- 事务隔离级别：READ_COMMITTED

#### 关键技术点示例

**数据库优化**：
- **索引优化**：为 WHERE 条件添加索引
- **SQL优化**：避免 SELECT *，只查询需要的字段
- **分页优化**：使用 LIMIT 和 OFFSET
- **批量操作**：使用批量插入、批量更新

#### 缓存策略示例

| 缓存类型 | Key设计 | 过期时间 | 更新策略 |
|---------|---------|---------|---------|
| 商品列表缓存 | `product:list:{page}:{pageSize}` | 5分钟 | 主动失效 |
| 用户信息缓存 | `user:info:{userId}` | 30分钟 | 更新即刷新 |
| 商品详情缓存 | `product:detail:{productId}` | 1小时 | 定时刷新 |
| 热点数据预加载 | `hot:products` | 10分钟 | 定时更新 |

#### 消息队列使用示例

**Kafka Topic 设计**:
- `order-created`: 订单创建事件
- `order-paid`: 订单支付事件
- `order-cancelled`: 订单取消事件
- `inventory-updated`: 库存更新事件

**消息格式**:
```json
{
  "eventId": "evt-20260121-00001",

  }

```

#### 安全机制示例

- **XSS防护**：输入过滤、输出转义

- **API限流**：基于用户+IP的限流

#### 异常处理示例

**全局异常处理**：

- 异常时自动回滚
