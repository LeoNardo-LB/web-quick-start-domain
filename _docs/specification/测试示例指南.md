## 测试原则

### 1. 测试金字塔

```
        /\
       /  \      E2E测试（少量）
      /----\     集成测试（适量）
     /------\    单元测试（大量）
    /--------\
   /__________\
```

- **单元测试**：最多，快速，隔离（纯Mock）
- **集成测试**：适量，真实环境（Spring + H2）
- **E2E测试**：最少，慢，端到端

### 2. 测试命名规范

```java
// ✅ 好的命名
void testCreateOrder_success()
void testCreateOrder_whenCustomerIdInvalid_thenThrowException()

// ❌ 避免的命名
void test1()
void testOrder()
```

### 3. Given-When-Then模式

```java
@Test
@DisplayName("创建订单成功")
void testCreateOrder_success() {
    // Given: 准备测试数据
    String customerId = "CUST001";
    Money totalAmount = new Money(100.00, "CNY");

    // When: 执行被测试方法
    OrderAggr order = OrderAggr.create(customerId, totalAmount);

    // Then: 验证结果
    assertThat(order.getCustomerId()).isEqualTo(customerId);
    assertThat(order.getStatus()).isEqualTo(OrderStatus.CREATED);
}
```

## 测试基类

### UnitTestBase（单元测试基类）

**特点**：
- 纯Mock测试，不启动Spring
- 0毫秒启动，完全隔离
- 适合测试业务逻辑

**位置**：`test/unit/org/smm/archetype/test/base/UnitTestBase.java`

```java
public abstract class UnitTestBase {
    // 自动注入Mock字段
    @Mock
    protected OrderRepository orderRepository;

    @BeforeEach
    void setUp() {
        // 初始化Mock（可选）
        MockitoAnnotations.openMocks(this);
    }

    @AfterEach
    void resetAllMocks() {
        // 重置所有Mock
        Mockito.reset();
    }
}
```

### IntegrationTestBase（集成测试基类）

**特点**：
- 完整Spring上下文
- H2内存数据库
- DBUnit数据预加载

**位置**：`test/integration/org/smm/archetype/test/base/IntegrationTestBase.java`

```java
@SpringBootTest
@TestExecutionListeners({
    DbUnitTestExecutionListener.class
})
@DatabaseSetup("/dbunit/order-data.xml")  // 预加载数据
public abstract class IntegrationTestBase {
    @Autowired
    protected OrderRepository orderRepository;
}
```

## Controller测试示例

### 测试目标

验证Controller层：
- 请求参数验证
- 返回值格式
- 异常处理
- HTTP状态码

### 完整示例

```java
package org.smm.archetype.adapter._example.order.web.api;

import com.fasterxml.jackson.databind.ObjectMapper;
import org.junit.jupiter.api.DisplayName;
import org.junit.jupiter.api.Test;
import org.smm.archetype.app._example.order.OrderAppService;
import org.smm.archetype.domain._example.order.model.aggregateroot.OrderAggr;
import org.smm.archetype.domain._shared.base.Response;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.boot.test.autoconfigure.web.servlet.WebMvcTest;
import org.springframework.boot.test.mock.mockito.MockBean;
import org.springframework.http.MediaType;
import org.springframework.test.web.servlet.MockMvc;

import static org.mockito.ArgumentMatchers.any;
import static org.mockito.Mockito.when;
import static org.springframework.test.web.servlet.request.MockMvcRequestBuilders.get;
import static org.springframework.test.web.servlet.result.MockMvcResultMatchers.jsonPath;
import static org.springframework.test.web.servlet.result.MockMvcResultMatchers.status;

/**
 * 订单Controller单元测试
 */
@WebMvcTest(OrderController.class)
@DisplayName("订单API单元测试")
class OrderControllerTest extends UnitTestBase {

    @Autowired
    private MockMvc mockMvc;

    @Autowired
    private ObjectMapper objectMapper;

    @MockBean
    private OrderAppService orderAppService;

    @Test
    @DisplayName("查询客户订单列表 - 成功")
    void testGetOrdersByCustomer_success() throws Exception {
        // Given: 准备Mock数据
        String customerId = "CUST001";
        OrderAggr order1 = OrderAggr.create(customerId, new Money(100.00, "CNY"));
        OrderAggr order2 = OrderAggr.create(customerId, new Money(200.00, "CNY"));

        when(orderAppService.getOrdersByCustomer(customerId))
            .thenReturn(List.of(order1, order2));

        // When & Then: 执行请求并验证
        mockMvc.perform(get("/api/orders")
                .param("customerId", customerId)
                .contentType(MediaType.APPLICATION_JSON))
            .andExpect(status().isOk())
            .andExpect(jsonPath("$.code").value(200))
            .andExpect(jsonPath("$.message").value("success"))
            .andExpect(jsonPath("$.data").isArray())
            .andExpect(jsonPath("$.data.length()").value(2))
            .andExpect(jsonPath("$.data[0].customerId").value(customerId));
    }

    @Test
    @DisplayName("创建订单 - 成功")
    void testCreateOrder_success() throws Exception {
        // Given
        String requestBody = """
            {
                "customerId": "CUST001",
                "totalAmount": {
                    "amount": 100.00,
                    "currency": "CNY"
                }
            }
            """;

        when(orderAppService.createOrder(any()))
            .thenReturn(OrderId.generate());

        // When & Then
        mockMvc.perform(post("/api/orders")
                .contentType(MediaType.APPLICATION_JSON)
                .content(requestBody))
            .andExpect(status().isOk())
            .andExpect(jsonPath("$.code").value(200));
    }

    @Test
    @DisplayName("查询订单 - 参数缺失")
    void testGetOrder_missingParameter() throws Exception {
        // When & Then
        mockMvc.perform(get("/api/orders"))
            .andExpect(status().isBadRequest());
    }
}
```

### 关键要点

1. **使用@WebMvcTest**：只加载Controller层，不启动完整Spring上下文
2. **MockMvc**：模拟HTTP请求
3. **@MockBean**：Mock依赖的Service
4. **jsonPath验证**：验证JSON响应格式

## ApplicationService测试示例

### 测试目标

验证ApplicationService层：
- 用例编排逻辑
- DTO转换
- 事务边界
- 异常处理

### 完整示例

```java
package org.smm.archetype.app._example.order;

import org.junit.jupiter.api.DisplayName;
import org.junit.jupiter.api.Test;
import org.junit.jupiter.api.extension.ExtendWith;
import org.mockito.InjectMocks;
import org.mockito.Mock;
import org.mockito.junit.jupiter.MockitoExtension;
import org.smm.archetype.domain._example.order.model.aggregateroot.OrderAggr;
import org.smm.archetype.domain._example.order.model.valueobject.OrderId;
import org.smm.archetype.domain._example.order.model.valueobject.Money;
import org.smm.archetype.domain._example.order.repository.OrderAggrRepository;
import org.smm.archetype.domain._example.order.service.command.CreateOrderCommand;
import org.smm.archetype.domain._example.order.service.query.OrderQuery;

import java.util.List;

import static org.assertj.core.api.Assertions.assertThat;
import static org.mockito.ArgumentMatchers.any;
import static org.mockito.Mockito.*;

/**
 * 订单应用服务单元测试
 */
@ExtendWith(MockitoExtension.class)
@DisplayName("订单应用服务单元测试")
class OrderAppServiceTest extends UnitTestBase {

    @Mock
    private OrderAggrRepository orderRepository;

    @InjectMocks
    private OrderAppService orderAppService;

    @Test
    @DisplayName("创建订单 - 成功")
    void testCreateOrder_success() {
        // Given
        CreateOrderCommand command = new CreateOrderCommand(
            "CUST001",
            new Money(100.00, "CNY")
        );

        when(orderRepository.save(any(OrderAggr.class)))
            .thenReturn(OrderId.generate());

        // When
        OrderId orderId = orderAppService.createOrder(command);

        // Then
        assertThat(orderId).isNotNull();
        verify(orderRepository, times(1)).save(any(OrderAggr.class));
    }

    @Test
    @DisplayName("查询客户订单 - 成功")
    void testGetOrdersByCustomer_success() {
        // Given
        String customerId = "CUST001";
        OrderAggr order1 = OrderAggr.create(customerId, new Money(100.00, "CNY"));
        OrderAggr order2 = OrderAggr.create(customerId, new Money(200.00, "CNY"));

        when(orderRepository.findByCustomerId(customerId))
            .thenReturn(List.of(order1, order2));

        // When
        OrderQuery query = new OrderQuery(customerId);
        List<OrderDTO> orders = orderAppService.getOrdersByCustomer(query);

        // Then
        assertThat(orders).hasSize(2);
        assertThat(orders.get(0).getCustomerId()).isEqualTo(customerId);
        verify(orderRepository, times(1)).findByCustomerId(customerId);
    }

    @Test
    @DisplayName("创建订单 - 仓储异常")
    void testCreateOrder_repositoryException() {
        // Given
        CreateOrderCommand command = new CreateOrderCommand(
            "CUST001",
            new Money(100.00, "CNY")
        );

        when(orderRepository.save(any(OrderAggr.class)))
            .thenThrow(new RuntimeException("Database error"));

        // When & Then
        assertThatThrownBy(() -> orderAppService.createOrder(command))
            .isInstanceOf(RuntimeException.class)
            .hasMessageContaining("Database error");
    }
}
```

### 关键要点

1. **纯Mock测试**：不启动Spring，只测试业务逻辑
2. **@ExtendWith(MockitoExtension.class)**：启用Mockito
3. **@InjectMocks + @Mock**：自动注入Mock依赖
4. **verify()**：验证Mock方法调用次数

## DomainService测试示例

### 测试目标

验证DomainService层：
- 业务规则
- 值对象逻辑
- 领域事件发布

### 完整示例

```java
package org.smm.archetype.domain._example.order.service;

import org.junit.jupiter.api.DisplayName;
import org.junit.jupiter.api.Test;
import org.smm.archetype.domain._example.order.model.aggregateroot.OrderAggr;
import org.smm.archetype.domain._example.order.model.enumeration.OrderStatus;
import org.smm.archetype.domain._example.order.model.valueobject.Money;
import org.smm.archetype.domain._example.order.service.specification.OrderSpecification;

import static org.assertj.core.api.Assertions.assertThat;
import static org.assertj.core.api.Assertions.assertThatThrownBy;

/**
 * 订单领域服务单元测试
 */
@DisplayName("订单领域服务单元测试")
class OrderDomainServiceTest extends UnitTestBase {

    @Test
    @DisplayName("计算订单总价 - 成功")
    void testCalculateTotalAmount_success() {
        // Given
        OrderAggr order = OrderAggr.create("CUST001", new Money(0, "CNY"));
        order.addItem("PROD001", new Money(100.00, "CNY"), 2);
        order.addItem("PROD002", new Money(50.00, "CNY"), 1);

        // When
        Money total = OrderDomainService.calculateTotalAmount(order);

        // Then
        assertThat(total.getAmount()).isEqualByComparingTo("250.00");
    }

    @Test
    @DisplayName("验证订单可支付 - 可支付")
    void testValidateOrderPayable_payable() {
        // Given
        OrderAggr order = OrderAggr.create("CUST001", new Money(100.00, "CNY"));

        // When & Then
        assertThat(OrderSpecification.payable().test(order)).isTrue();
    }

    @Test
    @DisplayName("验证订单可支付 - 不可支付")
    void testValidateOrderPayable_notPayable() {
        // Given
        OrderAggr order = OrderAggr.create("CUST001", new Money(100.00, "CNY"));
        order.pay(); // 状态变为PAID

        // When & Then
        assertThat(OrderSpecification.payable().test(order)).isFalse();
    }

    @Test
    @DisplayName("支付订单 - 成功")
    void testPayOrder_success() {
        // Given
        OrderAggr order = OrderAggr.create("CUST001", new Money(100.00, "CNY"));

        // When
        order.pay();

        // Then
        assertThat(order.getStatus()).isEqualTo(OrderStatus.PAID);
        assertThat(order.getDomainEvents())
            .hasSize(2)
            .anySatisfy(event -> assertThat(event.getEventTypeName())
                .isEqualTo("OrderPaid"));
    }

    @Test
    @DisplayName("支付订单 - 状态错误")
    void testPayOrder_invalidStatus() {
        // Given
        OrderAggr order = OrderAggr.create("CUST001", new Money(100.00, "CNY"));
        order.pay(); // 已支付

        // When & Then
        assertThatThrownBy(() -> order.pay())
            .isInstanceOf(IllegalStateException.class)
            .hasMessageContaining("只有已创建的订单可以支付");
    }
}
```

### 关键要点

1. **测试业务规则**：如金额计算、状态转换
2. **测试领域事件**：验证事件发布
3. **测试规格模式**：验证业务规则组合
4. **无Mock**：领域模型不需要Mock

## Repository测试示例

### 测试目标

验证Repository层：
- DO转换
- SQL执行
- 异常处理

### 完整示例

```java
package org.smm.archetype.infrastructure._example.order.repository;

import com.mybatisflex.core.query.QueryWrapper;
import org.junit.jupiter.api.DisplayName;
import org.junit.jupiter.api.Test;
import org.smm.archetype.domain._example.order.model.aggregateroot.OrderAggr;
import org.smm.archetype.domain._example.order.model.valueobject.Money;
import org.smm.archetype.domain._example.order.model.valueobject.OrderId;
import org.smm.archetype.infrastructure._example.order.repository.mapper.OrderAggrMapper;
import org.smm.archetype.infrastructure._shared.generated.repository.entity.OrderAggrDO;
import org.smm.archetype.test.base.IntegrationTestBase;
import org.springframework.beans.factory.annotation.Autowired;

import static org.assertj.core.api.Assertions.assertThat;

/**
 * 订单仓储集成测试
 */
@DisplayName("订单仓储集成测试")
class OrderAggrRepositoryImplTest extends IntegrationTestBase {

    @Autowired
    private OrderAggrRepository orderRepository;

    @Autowired
    private OrderAggrMapper orderAggrMapper;

    @Test
    @DisplayName("保存订单 - 成功")
    void testSave_success() {
        // Given
        OrderAggr order = OrderAggr.create("CUST001", new Money(100.00, "CNY"));

        // When
        orderRepository.save(order);

        // Then
        OrderAggrDO savedDO = orderAggrMapper.selectOneById(order.getId().getValue());
        assertThat(savedDO).isNotNull();
        assertThat(savedDO.getCustomerId()).isEqualTo("CUST001");
    }

    @Test
    @DisplayName("根据ID查询订单 - 成功")
    void testFindById_success() {
        // Given
        OrderAggr order = OrderAggr.create("CUST001", new Money(100.00, "CNY"));
        orderRepository.save(order);

        // When
        OrderAggr found = orderRepository.findById(order.getId());

        // Then
        assertThat(found).isNotNull();
        assertThat(found.getCustomerId()).isEqualTo("CUST001");
    }

    @Test
    @DisplayName("根据ID查询订单 - 不存在")
    void testFindById_notFound() {
        // When
        OrderAggr found = orderRepository.findById(OrderId.generate());

        // Then
        assertThat(found).isNull();
    }

    @Test
    @DisplayName("根据客户ID查询订单 - 成功")
    void testFindByCustomerId_success() {
        // Given
        OrderAggr order1 = OrderAggr.create("CUST001", new Money(100.00, "CNY"));
        OrderAggr order2 = OrderAggr.create("CUST001", new Money(200.00, "CNY"));
        orderRepository.save(order1);
        orderRepository.save(order2);

        // When
        List<OrderAggr> orders = orderRepository.findByCustomerId("CUST001");

        // Then
        assertThat(orders).hasSize(2);
    }
}
```

### 关键要点

1. **继承IntegrationTestBase**：启动Spring + H2数据库
2. **真实Mapper**：测试SQL执行和DO转换
3. **DBUnit数据预加载**：准备测试数据
4. **事务自动回滚**：测试数据不污染数据库

## 常见测试场景

### 1. 测试异常

```java
@Test
@DisplayName("参数验证失败")
void testValidate_invalidParameter() {
    // Given
    String emptyCustomerId = "";

    // When & Then
    assertThatThrownBy(() -> OrderAggr.create(emptyCustomerId, money))

        .hasMessageContaining("客户ID不能为空");

```

### 2. 测试Mock验证

```java
@Test
@DisplayName("验证Mock调用")
void testMockVerification() {
    // When
    orderAppService.createOrder(command);

    // Then: 验证调用次数
    verify(orderRepository, times(1)).save(any());
    verify(orderRepository, never()).delete(any());
}
```

### 3. 测试异步方法

```java
@Test
@DisplayName("测试异步事件发布")
void testAsyncEventPublish() throws Exception {
    // When
    applicationEventPublisher.publishEvent(event);

    // Then: 等待异步完成
    await().atMost(1, TimeUnit.SECONDS)
        .untilAsserted(() -> {
            verify(eventHandler, times(1)).handle(event);
        });
}
```

### 4. 测试数据库约束

```java
@Test
@DisplayName("测试唯一约束")
void testUniqueConstraint() {
    // Given: 插入第一条记录
    OrderAggr order1 = OrderAggr.create("CUST001", new Money(100.00, "CNY"));

    // Then
    assertThatThrownBy(() -> orderRepository.save(order2))
        .hasCauseInstanceOf(DuplicateKeyException.class);
}
```

## 测试最佳实践

### DO ✅

1. **独立测试**：每个测试独立运行，互不依赖
2. **快速测试**：单元测试<100ms，集成测试<5s
3. **清晰命名**：`test{Method}_{Scenario}_{ExpectedResult}`
4. **覆盖边界**：正常、边界、异常情况都要测试
5. **使用断言**：assertThat > assertTrue

### DON'T ❌

1. **测试私有方法**：通过公共接口测试
2. **测试第三方库**：信任Spring、MyBatis等
3. **过度Mock**：只Mock外部依赖
4. **忽略测试**：避免@Disabled或@Ignore
5. **复杂测试逻辑**：测试代码应该简单清晰

## 常见问题

### Q1: 如何Mock静态方法？

**A**: 使用Mockito的inline-mock-maker：

```java
// 1. 添加依赖
// test/pom.xml
// <dependency>
//   <groupId>org.mockito</groupId>
//   <artifactId>mockito-inline</artifactId>
// </dependency>

// 2. 使用MockedStatic
try (MockedStatic<UUID> mocked = mockStatic(UUID.class)) {
    mocked.when(UUID::randomUUID).thenReturn(uuid);
    // 测试代码
}
```

### Q2: 如何测试事务？

**A**: 使用@Transactional注解 + 自动回滚：

```java
@SpringBootTest
@Transactional  // 测试后自动回滚
class TransactionTest extends IntegrationTestBase {

    @Test
    @DisplayName("测试事务回滚")
    void testTransactionRollback() {
        // 测试代码
    }
}
```

### Q3: 如何测试定时任务？

**A**: 手动触发定时任务方法：

```java
@SpringBootTest
class ScheduledTaskTest {

    @Autowired
    private OrderRetryScheduledTask task;

    @Test
    @DisplayName("测试订单重试任务")
    void testRetryTask() {
        // When: 手动执行
        task.retryFailedOrders();

        // Then: 验证结果
    }
}
```
