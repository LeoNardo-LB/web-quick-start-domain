## 🎯 使用指南

### 按场景查找文档

#### 场景1：首次接触项目

1. 阅读 [验证流程指南](业务代码生成(AI)流程.md) - 了解TDD代码验证流程

#### 场景2：开始编写业务代码

1. 先读 [业务代码编写规范](业务代码编写规范.md) - 了解编码规范
2. 再读 [测试代码编写与示例指南](测试代码编写与示例指南.md) - 了解测试规范
3. 最后执行 [验证流程指南](业务代码生成(AI)流程.md) - 完成TDD代码验证

#### 场景3：编写测试代码

1. 阅读 [测试代码编写与示例指南](测试代码编写与示例指南.md) - 了解测试编写标准和查看完整示例

#### 场景4：验证代码质量

1. 严格按照 [验证流程指南](业务代码生成(AI)流程.md) - 执行TDD 5步验证流程
    - Phase 0：需求确认与TDD规划（反复询问用户）
    - Phase 1：编写单元测试（TDD红阶段）
    - Phase 2：编译验证（`mvn clean compile`）
    - Phase 3：执行单元测试（TDD绿阶段，`mvn test`）
    - Phase 4：启动验证（最关键，`mvn test -Dtest=ApplicationStartupTests -pl test`）

## 📖 各文档核心内容

### 业务代码生成(AI)流程.md

- **5步强制验证流程（TDD模式）**：
    - Phase 0：需求确认与TDD规划（反复询问用户）⭐ 新增
    - Phase 1：编写单元测试（TDD红阶段）
    - Phase 2：编译验证
    - Phase 3：执行单元测试（TDD绿阶段）
    - Phase 4：启动验证（最关键）
- **TDD红-绿-重构循环**：测试驱动开发流程
- **代码质量检查清单**：编译、测试、覆盖率要求
- **常见问题处理**：编译失败、测试失败、启动失败
- **循环依赖解决原则**：通过重构解耦，禁止使用@Lazy等
- **需求确认流程**：业务、技术、架构等多方面细节确认

### 业务代码编写规范.md

- **第1章**：基础编码规范（格式、命名、Lombok、日志）
- **第2章**：框架使用规范（SpringBoot Bean管理、Maven依赖）
- **第3章**：领域建模规范（枚举、聚合根、实体、值对象）
- **第4章**：数据持久化规范（DDL、字段类型、索引）
- **第5章**：分层开发规范（Adapter、Application、Domain、Infrastructure）
- **第6章**：设计原则（SOLID、接口-实现分离）
- **第7章**：中间件接入规范（缓存、事件、通知）
- **第8章**：禁止事项（代码、安全、性能、架构）
- **第9章**：测试规范（概述+引用）

### 测试代码编写与示例指南.md

- **项目测试结构**：test模块 + 测试基类
- **包名映射规则**：类名命名、包名对应关系
- **单元测试规范**：纯Mock，不启动Spring
- **集成测试规范**：Spring + H2 + DBUnit
- **测试数据管理**：DBUnit使用方法
- **测试基类使用**：UnitTestBase/IntegrationTestBase
- **各层测试示例**：
    - Controller测试示例（完整代码）
    - ApplicationService测试示例（完整代码）
    - DomainService测试示例（完整代码）
    - Repository测试示例（完整代码）
- **常见测试场景**：异常测试、Mock验证、异步测试、参数化测试等
- **测试最佳实践**：DO/DONT、常见问题FAQ
- **覆盖率要求**：行≥95%，分支100%
- **质量红线**：100%通过率，0次DDL修改
